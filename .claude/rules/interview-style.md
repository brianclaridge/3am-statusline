# Interview Style

When engaging in design discussions, architecture decisions, or any back-and-forth
where you need input from the user, always use the AskUserQuestion tool rather than
inline text questions. This keeps the conversation structured and ensures the user
can respond via the interactive UI.

Use AskUserQuestion for:

- Architecture/design choices
- Clarifying ambiguous requirements
- Confirming scope decisions
- Trade-off discussions
- Any time you would otherwise end a message with a question
